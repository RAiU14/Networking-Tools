# Networking Tools

A collection of Python-based network utilities for reachability checks, command-output capture, and vendor-agnostic EOL lifecycle lookup/caching.

This version is prepared for public or shared Git hosting: runtime logs, device captures, local spreadsheets, cache folders, and credential-bearing files are ignored. The EOL dashboard now includes a visible Start Here workflow for first-time users.

## Project layout

```text
Networking-Tools/
├── Alive_Checks/            # Direct and jump-host reachability checks
├── Log_Capture/             # SSH command collection from device inventory files
├── Network_EOL_Manager/     # FastAPI/React Network EOL Manager
├── docs/                    # Security, cleanup, and publishing guidance
├── CONTRIBUTING.md
├── LICENSE
└── README.md
```

## Tools

### Alive Checks

Checks whether a device is reachable using local ping or a configured jump host.

```bash
cd Alive_Checks
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python alive.py 192.0.2.10
```

Jump-host credentials must be provided locally through environment variables or function arguments. Do not hardcode them.

### Log Capture

Connects to devices with Netmiko, runs a command list, and writes local command-output captures plus CSV/JSON run summaries.

```bash
cd Log_Capture
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp Sheets/devices.example.csv Sheets/devices.csv
cp Sheets/commands.example.csv Sheets/commands.csv
python command_run.py --devices Sheets/devices.csv --commands Sheets/commands.csv --output-dir outputs --prompt-password
```

`devices.csv`, `commands.csv`, captures, logs, reports, and failed-device files are ignored by Git. You can also set `NETTOOLS_SSH_PASSWORD` instead of storing passwords in the spreadsheet.

### Network EOL Manager

Self-hosted network hardware lifecycle cache, API, and guided dashboard for internal inventory planning (currently supports Cisco EOL lookups).

```bash
cd Network_EOL_Manager
cp .env.example .env
docker compose up -d --build
```

Open locally:

```text
GUI:      http://127.0.0.1:5173
API docs: http://127.0.0.1:8000/docs
GraphQL:  http://127.0.0.1:8000/graphql
Health:   http://127.0.0.1:8000/health
```

Use the GUI Start Here panel for the normal path: create a database, run PID lookup, review evidence, then export reports. See `Network_EOL_Manager/README.md` for full setup, security, API, GraphQL, export, database, and legal notes.

## Security rules

Never commit:

- Device inventories or spreadsheets with hostnames, IPs, usernames, or passwords.
- SSH passwords, API tokens, client secrets, private keys, or `.env` files.
- Device command captures, failed-device reports, runtime logs, local databases, or generated cache files.
- Vendor datasets generated by your local environment.

Use `.env.example` files and `*.example.csv` files for documentation only. Keep real values in local untracked files.

More details:

- `docs/SECURITY.md`
- `docs/REPO_CLEANUP_REPORT.md`
- `docs/PUBLISHING_CHECKLIST.md`

## Basic validation

From the repository root:

```bash
python -m compileall Alive_Checks Log_Capture Network_EOL_Manager/backend Network_EOL_Manager/tools
pytest -q Network_EOL_Manager/tests
```

For Network EOL Manager backend tests:

```bash
cd Network_EOL_Manager
python -m venv .venv
source .venv/bin/activate
pip install -r requirements-dev.txt
pytest -q
```

For the frontend:

```bash
cd Network_EOL_Manager/front_end
npm install
npm run build
```
